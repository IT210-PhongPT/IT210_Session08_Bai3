PHẦN 1 - BÁO CÁO PHÂN TÍCH (I/O) & THIẾT KẾ GIẢI PHÁP
---------------------------------------------------------

1. XÁC ĐỊNH INPUT / OUTPUT
----------------------------
INPUT:
  - withdrawAmount : Long (có thể là null nếu client không gửi)
  - Các giá trị nguy hiểm cần xử lý:
      * null           → bẫy dữ liệu 1
      * -50000         → bẫy dữ liệu 2 (số âm)
      * 55000          → không phải bội số 10.000
      * 49000          → nhỏ hơn mức tối thiểu 50.000
      * 100000         → hợp lệ ✓

OUTPUT (per field):
  - PASS : không có lỗi, cho phép xử lý tiếp
  - FAIL : trả về message lỗi tương ứng, HTTP 400

2. TẠI SAO KHÔNG DÙNG @Min + if-else?
----------------------------------------
  - @Min(50000) giải quyết được điều kiện 1 nhưng KHÔNG thể
    kiểm tra "bội số của 10.000" → cần logic tùy biến.
  - if-else trong Controller vi phạm Single Responsibility Principle:
    Controller chỉ nên điều phối request, không chứa business logic.
  - Custom Annotation tách biệt hoàn toàn, tái sử dụng được ở
    bất kỳ DTO nào khác trong toàn hệ thống.

3. THIẾT KẾ GIẢI PHÁP: @MultipleOfTenThousand
------------------------------------------------
Tạo 1 Custom Annotation gộp CẢ 2 điều kiện vào 1 Validator:
  - Điều kiện 1: >= 50.000
  - Điều kiện 2: % 10.000 == 0

Áp dụng lên field withdrawAmount trong DTO.

4. PSEUDOCODE — LOGIC BÊN TRONG VALIDATOR
------------------------------------------

function isValid(value, context):

    # Bước 1: Xử lý null theo Best Practice Bean Validation
    # Null được coi là "không có giá trị" → trả về TRUE
    # Để bắt null, dùng @NotNull riêng ở DTO (phân tách trách nhiệm)
    IF value IS NULL:
        RETURN TRUE

    # Bước 2: Kiểm tra số âm hoặc bằng 0
    IF value <= 0:
        RETURN FALSE  → "Số tiền phải lớn hơn 0"

    # Bước 3: Kiểm tra mức tối thiểu 50.000
    IF value < 50_000:
        RETURN FALSE  → "Số tiền tối thiểu là 50.000 VNĐ"

    # Bước 4: Kiểm tra bội số của 10.000
    IF value % 10_000 != 0:
        RETURN FALSE  → "Số tiền phải là bội số của 10.000 VNĐ"

    # Tất cả điều kiện thoả mãn
    RETURN TRUE

  TÓM TẮT LUỒNG XỬ LÝ

withdrawAmount (Long)
        │
        ▼
